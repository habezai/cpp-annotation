2.2: C++的历史

C++的第一个实现是在20世纪80年代在AT&T贝尔实验室开发的，Unix操作系统就是在那里创建的。

C++ 最初是一个“预编译器”pre-compiler，类似于 C 的预处理器preprocessor，将源代码中的特殊结构转换为纯 C。当时，这段代码是由一个标准的 C 编译器编译的。由 C++ 预编译器读取的“ pre-code”通常位于扩展名为 .cc 、 .C 或 .cpp 的文件中。然后，这个文件将被转换为扩展名为 .c 的 C 源文件，该文件随后被编译并链接。

C++ 源文件的命名法仍然存在：扩展名.cc和.cpp仍使用。但是，如今通常在实际汇编过程中执行 C++ 预编译器的初步工作。编译器通常根据其扩展名确定源文件中使用的语言。这适用于 Borland 和 Microsoft 的 C++ 编译器，它们假定 C++ 源代码为 扩展名.cpp 。在许多 Unix 平台上可用的 GNU 编译器 g++ 假定 C++ 为 扩展名.cc 。

从 C++ 是 C 的超集这一事实也可以看出 C++ 曾经被编译成 C 代码的事实：C++ 提供了完整的 C 语法并支持所有 C 库函数，并添加了它自己的这些功能。这使得从 C 到 C++ 的转换非常容易。熟悉 C 的程序员可以通过使用扩展名为 .cc 或 .cpp 而不是 .c 的源文件开始“用 C++ 编程”，然后可以轻松地体验 C++ 提供的所有可能性。不需要突然改变习惯。
